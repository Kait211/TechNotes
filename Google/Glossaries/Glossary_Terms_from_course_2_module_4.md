**A record:** The most common DNS resource record, used to map a domain name to an IPv4 address.

**Anycast:** A routing technique that directs traffic to the best destination based on factors such as location, congestion, or network health.

**Automatic allocation:** A DHCP method where a permanent IP address is automatically assigned from a reserved range.

**Caching and recursive name servers:** DNS servers that store domain name lookups temporarily to speed up future requests. They are typically provided by an ISP or local network.

**CNAME:** A DNS resource record that maps one domain name (alias) to another domain name.

**DHCP discovery:** The process a DHCP client uses to obtain network configuration information from a DHCP server.

**Domain Name System (DNS):** A distributed service that translates domain names (such as `www.example.com`) into IP addresses.

**DNS zones:** Portions of the DNS namespace managed by an authoritative name server.

**Domain:** A section of the DNS hierarchy where control passes from a TLD server to an authoritative name server.

**Domain name:** A human-readable website name (e.g., `google.com`).

**Dynamic allocation:** A DHCP method that temporarily assigns an available IP address from a pool to a client device.

**Fixed allocation:** A DHCP method that assigns a specific IP address to a device based on its MAC address.

**Fully qualified domain name (FQDN):** The complete domain name, including all levels of the DNS hierarchy (e.g., `www.google.com`).

**IP masquerading:** A NAT technique that hides the sender's private IP address by replacing it with the router's public IP address.

**MX record:** A DNS resource record that specifies the mail server responsible for receiving email for a domain.

**Name resolution:** The process of using DNS to translate a domain name into an IP address.

**Network Address Translation (NAT):** A technique that translates private IP addresses to a public IP address, allowing multiple devices to share one public IP.

**NS record:** A DNS resource record that identifies the authoritative name servers for a DNS zone.

**NTP servers:** Servers that synchronize the date and time across computers on a network.

**Pointer (PTR) record:** A DNS resource record used for reverse DNS lookups by mapping an IP address to a domain name.

**Port forwarding:** A NAT technique that forwards traffic arriving on a specific port to a designated device on a private network.

**Port preservation:** A NAT technique where the router keeps the same source port selected by the client.

**Proxy service:** A server that acts as an intermediary between a client and another server.

**Quad A (AAAA) record:** A DNS resource record that maps a domain name to an IPv6 address.

**Recursive name servers:** DNS servers that perform the full process of resolving domain names on behalf of clients.

**Reverse lookup zone files:** DNS zone files that map IP addresses back to fully qualified domain names.

**Reverse proxy:** A server that receives client requests and forwards them to one or more backend servers while appearing as a single server.

**Round robin:** A load-balancing technique that distributes requests among multiple servers in sequence.

**SRV record:** A DNS resource record that specifies the location of a particular service.

**Start of Authority (SOA):** A DNS record that identifies the authoritative name server and contains administrative information for a DNS zone.

**Top-Level Domain (TLD):** The highest level of the DNS hierarchy, such as `.com`, `.org`, or `.edu`.

**Time-To-Live (TTL):** A value that determines how long DNS records are cached before they must be refreshed.

**Two-factor authentication (2FA):** A security method requiring two forms of verification, such as a password and a temporary code.

**TXT record:** A DNS resource record used to store text information, commonly for domain verification and email security.

**Types of DNS servers:** The five main DNS server types are **caching**, **recursive**, **root**, **TLD**, and **authoritative** name servers.

**Virtual Private Network (VPN):** A technology that securely extends a private network over the internet by creating an encrypted connection.

**Zone files:** Configuration files that contain all DNS resource records for a particular DNS zone.
